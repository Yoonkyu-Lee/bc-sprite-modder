# 캔버스 에디터 후보 비교 (선택 가이드)

올인원 스프라이트 모더에 **캔버스가 있는 에디터**를 어디서 가져올지 정하기 위한 비교표와 선택 기준.

---

## 1. 한눈에 보는 비교

| 항목 | Pixcil | Piskel | Pixel Paint Website | LibreSprite |
|------|--------|--------|---------------------|-------------|
| **언어/플랫폼** | Rust → WASM/데스크톱 | JS/TS, 웹 | HTML/CSS/JS | C++, 데스크톱 |
| **라이선스** | Apache-2.0 | Apache-2.0 | (저장소 확인) | GPL-2.0 |
| **기능 수** | 보통 (캔버스·레이어·프레임) | 많음 (애니·내보내기 등) | **최소** | 매우 많음 (Aseprite급) |
| **코드베이스** | 중간, 모듈화됨 | 큼 | **매우 작음** | 매우 큼 |
| **600×800 성능** | **우수** (Rust/WASM) | 양호 | 확장 후 보통 | 우수 |
| **우리 앱에 넣는 방식** | WASM 번들 → WebView/Electron | 웹 빌드 → WebView/Electron | iframe/WebView 또는 **코드 복사** | **별도 실행** + 파일 연동 |
| **뜯어고치기 난이도** | 중간 (Rust 수정) | 중간 (TS, 구조 파악 필요) | **쉬움** (파일 적음) | 어려움 (C++, 빌드 무거움) |
| **실시간 뷰어 연동** | WASM ↔ JS 버퍼 공유 가능 | 같은 페이지에서 버퍼 공유 가능 | 같은 페이지에서 버퍼 공유 가능 | IPC/파일만 가능 |

## 2. “우선순위별”로 고르기

### A. “기능은 적당히, 성능·최적화가 최우선”  
→ **Pixcil**  
- Rust 코어라 600×800 같은 크기에서 안정적.  
- 레이어·프레임 있음. 웹은 WASM이라 PySide6 앱에서는 WebView에 넣으면 됨.  
- 단점: Rust 빌드/수정이 필요하면 진입 장벽 있음.

### B. “코드베이스 작고, 우리가 전부 통제하고 싶음”  
→ **Pixel Paint Website**  
- 파일 수 적고, 캔버스 크기·저장 경로만 바꿔서 우리 쪽에 맞추기 쉽다.  
- 기능은 우리가 레이어·툴 조금씩 추가하는 식으로 확장.  
- 단점: “원래부터 대형 캔버스 최적화”는 아니어서, 600×800에서 부담되면 그리기/합성 부분만 우리가 손봐야 함.

### C. “에디터 기능은 이미 다 있고, 연동만 하면 됨”  
→ **Piskel**  
- 캔버스·레이어·애니·내보내기 다 있음. Electron/WebView에 넣고, 저장만 `mods/` 쪽으로 바꾸면 됨.  
- 실시간 반영은 같은 페이지에서 ImageData/텍스처 공유하면 됨.  
- 단점: 코드베이스 커서 “일부만 쓰기”나 “UI 대폭 단순화”는 작업량이 있음.

### D. “에디터는 따로 돌리고, 앱은 데이터·리패킹만”  
→ **LibreSprite**  
- 에디터는 LibreSprite 독립 실행, 우리 앱은 Workspace·프로젝트·데이터 준비·리패킹만 담당.  
- “mods/ 여기 저장해라”만 지정해 주고, 리패킹 시 그 폴더만 읽음.  
- 단점: “앱 안에 캔버스가 붙어 있는” 느낌은 없고, 실시간 애니 뷰어 연동도 파일/수동 리로드 수준.

---

## 3. 혼란을 줄이는 질문 두 개

1. **“캔버스가 우리 앱 **안**에 붙어 있어야 하나, **밖**에 있어도 되나?”**  
   - 안: Pixcil(WASM) / Piskel(웹) / Pixel Paint(웹 또는 복사) 중 하나를 WebView로 넣는 쪽.  
   - 밖: LibreSprite만 쓰고, 우리 앱은 경로·리패킹만.

2. **“에디터 코드를 **직접 수정**해서 우리 목적에 맞출 계획이 있나?”**  
   - 많이 수정: Pixel Paint(작은 코드) 또는 Piskel(필요 부분만).  
   - 최소 수정: Pixcil(WASM 그대로 넣고, 저장/로드 경로만 JS로 연결).  
   - 수정 거의 없음: LibreSprite 별도 실행.

---

## 4. 추천 조합 (상황별)

| 상황 | 추천 |
|------|------|
| 메인 앱이 **Python + PySide6**이고, **앱 안에** 캔버스 넣고 싶음 | **Piskel** 또는 **Pixcil WASM** 을 QWebEngineView에 로드. 저장/버퍼만 앱과 연동. |
| 메인 앱이 **Electron** (이미 웹 기술) | **Piskel** 소스를 우리 레포에 넣고, 탭/저장 경로만 수정. |
| **가장 단순한 출발** (기능은 나중에 추가) | **Pixel Paint Website** 포크 → 캔버스 크기 600×800, 저장을 `mods/` 로. |
| **성능·품질 최우선**, Rust/WASM 괜찮음 | **Pixcil** 웹 빌드를 앱에 임베드. |
| **에디터는 기존 툴 쓰고**, 우리는 파이프라인만 | **LibreSprite** 별도 실행 + 우리 앱에서 `mods/` 경로만 관리. |

---

정리하면, “고민된다”는 건 **안에 넣을지 / 밖에 둘지**, **직접 수정할지 / 최소 연동만 할지**가 아직 안 정해져서일 수 있음.  
위에서 1번·2번 질문에 답한 뒤, 4번 표에서 상황에 맞는 한 줄만 골라서 그걸로 진행해 보는 걸 추천함.

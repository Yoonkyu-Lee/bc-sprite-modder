# 개발 관점 계획서 (Development Perspective)

USER_PERSPECTIVE.md의 유저 플로우를 구현하기 위한 개발 단계·기술 결정·산출물 정리.

---

## 1. 목표 및 범위

- **목표:** 냥코대전쟁 스프라이트 모딩 올인원 도구  
  `Workspace 선택 → 데이터 셋 준비 → 프로젝트 생성 → 스프라이트 탐색/편집 → 리패킹` 을 하나의 앱에서 수행.
- **범위:** 데스크톱 단일 실행 파일(또는 Python + Qt 실행 환경). 데이터(BCData, projects)는 사용자 지정 Workspace 아래에만 생성.

---

## 2. 기술 스택 제안

| 구분 | 선택 | 비고 |
|------|------|------|
| **메인 앱** | Python 3.10+ / PySide6 | 기존 data_setup.py, mod_repack.py, tbcml과 동일 런타임으로 통합 용이. |
| **데이터/로직** | 프로젝트 내 tbcml 소스 (tbcml/src) | APK 로드, 추출, GamePacks, 리패킹은 기존 스크립트 호출 또는 모듈 import. |
| **UI 구조** | PySide6 단일 윈도우 + 탭 또는 도킹 | [데이터 준비] [프로젝트] [스프라이트 편집] [리패킹] 탭. 필요 시 QDockWidget/QSplitter. |
| **애니메이션 뷰어** | 캐시 기반 렌더링 | 프레임별 보간 결과를 미리 계산해 두고, 재생 시 캐시만 참조 (state.rs 방식). QGraphicsView 또는 OpenGL 위젯. |
| **캔버스 에디터** | 임베드 선택지 (1) Pixcil WASM (2) Piskel 커스텀 빌드 (3) Qt 캔버스 직접 구현 | 600×800 전후 크기, 레이어·기본 툴 필요. 실시간 반영은 공유 버퍼/텍스처로 처리. |
| **설정 저장** | Workspace 경로, 최근 프로젝트 목록 | 로컬 설정 파일 (예: `~/.config/BattleCatsSpriteLab/settings.json` 또는 Workspace 내부 `.lab/settings.json`). |

## 3. 디렉터리/데이터 구조

- **Workspace 루트** (사용자 지정)
  - `BCData/`  
    - `apks/<package>/<version>/` (원본/추출/서버 데이터) — 기존 tbcml 구조 유지.
  - `projects/`  
    - `<프로젝트명>/mods/`  
    - `<프로젝트명>/exports/`  
    - `<프로젝트명>/project.json`
  - (선택) `.lab/settings.json` — 앱이 사용하는 Workspace 단 설정.

- **project.json 스키마 (안)**  
  - `name`, `description`, `created`, `modified`  
  - `dataset`: `{ "server": "kr", "version": "15.1.0", "path": "BCData/apks/..." }`  
  - (선택) `last_opened_sprite`, `window_geometry`

## 4. 개발 단계 (Phase)

### Phase 0: 부트스트랩 및 Workspace

- **산출물**
  - 앱 진입점 (예: `main.py` + PySide6 `QApplication`).
  - “Workspace 선택” 다이얼로그 (기본값: `문서/BattleCatsSpriteLab`).
  - Workspace 선택 시 `BCData/`, `projects/` 생성.
  - 선택한 Workspace 경로 저장 (설정 파일 또는 레지스트리/플랫폼별).
  - 재실행 시 마지막 Workspace 자동 로드.
- **의존성:** PySide6, pathlib, json.

---

### Phase 1: [데이터 준비] 탭

- **기능**
  - 서버 선택 (한국/일본 등), 게임 버전 선택 (목록 + 최신 버전 표시).
  - APK: “자동 다운로드” 또는 “파일 지정”.
  - 진행: 다운로드 → 추출 → 서버 데이터 다운로드 → 팩 검사.
  - 로그/진행 상태를 탭 내에 표시.
  - 완료 시 “이 데이터 셋으로 새 프로젝트 만들기” 버튼 → [프로젝트] 탭으로 전환 + 데이터 셋 컨텍스트 전달.
- **구현**
  - data_setup.py 로직을 함수 단위로 호출 (또는 모듈 import 후 `run_data_setup` 등).
  - 스레드/스레드 풀에서 실행하고, 시그널/슬롯으로 UI 업데이트 (진행 바, 로그 텍스트).
- **산출물:** 데이터 준비 탭 UI, tbcml 연동, 진행/로그 표시.

---

### Phase 2: [프로젝트] 탭

- **기능**
  - 프로젝트 이름/설명 입력 폼.
  - 사용할 데이터 셋 선택 (Phase 1에서 준비된 목록 또는 BCData 스캔 결과).
  - “생성” 시 `projects/<이름>/mods/`, `exports/`, `project.json` 생성.
  - 프로젝트 리스트 (테이블 또는 리스트), 더블클릭 시 “현재 작업 프로젝트”로 설정하고 [스프라이트 편집] 등으로 이동.
  - 최근 프로젝트 목록 표시 (설정 또는 project.json 수정 시각 기준).
- **구현**
  - project.json 읽기/쓰기, 데이터 셋 메타는 BCData 디렉터리 스캔 또는 별도 인덱스 파일.
- **산출물:** 프로젝트 탭 UI, 프로젝트 CRUD, 현재 프로젝트 상태 보관 (앱 전역 또는 설정).

---

### Phase 3: [스프라이트 편집] — 브라우저 (유닛·폼 기반)

- **개념:** 탐색 단위는 "스프라이트 파일"이 아니라 **"캐릭터(유닛 ID) + 폼(f/c/s/u)"**. e(광란)는 별도 캐릭터.
- **기능**
  - **1단계:** 유닛 목록 (ID 또는 이름). 광란(e)은 별도 그룹/필터.
  - **2단계:** 선택한 유닛의 폼 목록 (실제 존재하는 f/c/s/u만 표시).
  - **3단계:** 선택한 (유닛, 폼)에 속한 에셋 목록 (imgcut, mamodel, maim, PNG 등). 항목 선택 시 편집 화면 진입.
  - 검색/필터: 유닛 ID, 폼 코드, 파일명.
- **구현**
  - GamePacks 스캔으로 (unit_id, form) 쌍 수집: ImageDataLocal 등에서 `{id}_{form}.imgcut`, `uni{id}_{form}*.png` 등 패턴으로 유닛·폼 인덱스 구축.
  - 좌측(또는 상단): 유닛 목록 → 폼 선택 → 에셋 리스트. 선택 시 file_name + pack_name + (unit_id, form) 컨텍스트 전달.
  - 썸네일: 해당 폼의 대표 이미지(아이콘 또는 첫 프레임); 캐시 권장.
- **산출물:** 유닛·폼·에셋 3단계 브라우저 UI, GamePacks 연동, 유닛/폼 인덱스 로직.

---

### Phase 4a: [스프라이트 편집] — 편집 화면 진입 및 뷰어

- **기능**
  - 브라우저에서 **에셋**(유닛·폼 컨텍스트 내 한 파일) 선택 시 **편집 화면**으로 전환.
  - **정적 뷰어:** 선택한 스프라이트 이미지 표시 (GamePacks.get_img 등). 단일 PNG면 그대로, 애니메이션(maanim/imgcut)이면 프레임 목록만 표시해 **프레임 선택 시 해당 프레임 이미지** 표시.
  - 우측(또는 우상단): **프레임/파트 리스트** (클릭 시 뷰어에 해당 프레임 표시).
  - (선택) 재생/일시정지, 프레임 슬라이더 — 단순 구현 가능 범위.
- **구현**
  - 유닛·폼·에셋 선택 시 (file_name, pack_name, unit_id, form) 전달. 편집 패널은 file_name 기준으로 get_img/get_sprite_frames 호출.
  - tbcml BCImage / imgcut·mamodel 파싱으로 프레임 목록 확보, 첫 프레임 또는 선택 프레임을 QLabel/QGraphicsView로 렌더.
  - **저장:** “저장” 시 `projects/<프로젝트>/mods/<파일명>.png` 에 쓰고, 필요 시 GamePacks 쪽 메모리도 동기화.
- **산출물:** 편집 화면 레이아웃, 정적(또는 단순 프레임 선택) 뷰어, 프레임/파트 리스트.

---

### Phase 4b: [스프라이트 편집] — 캔버스 에디터 및 mods/ 저장

- **기능**
  - **캔버스 에디터:** 현재 선택한 프레임(또는 단일 이미지)을 편집. 그리기/지우기, 외부 PNG 임포트, 투명도, 격자, 미러 등 기본 툴.
  - **저장:** "저장" 시 `projects/<프로젝트>/mods/<파일명>.png` 에 쓰기. GamePacks.set_file 호출로 메모리 동기화(선택).
- **구현**
  - 캔버스: (A) Pixcil WASM / (B) Piskel 임베드 / (C) Qt QGraphicsScene + 툴 바 중 선택. 편집 결과는 QImage/메모리 버퍼로 보관.
  - 저장 시 프로젝트 mods/ 경로에 PNG 쓰기, 필요 시 tbcml 포맷(암호화)으로 set_file.
- **산출물:** 캔버스 UI, 툴 바, mods/ 저장 플로우.

---

### Phase 4c: [스프라이트 편집] — 실시간 미리보기 및 애니메이션 뷰어 고도화

- **기능**
  - **실시간 미리보기:** 편집 버퍼가 갱신되면 뷰어가 해당 파츠만 반영해 **즉시 다시 그리기** (디스크 저장 없이 메모리 공유).
  - **애니메이션 뷰어 고도화:** 프레임별 보간 캐시 + 고정 타임스텝(예: 30fps), 재생/일시정지, 속도, 프레임 슬라이더. 편집 중인 파츠 텍스처만 교체해 재생.
- **구현**
  - 편집 결과를 메모리 버퍼(RgbaImage/QImage)로 노출하고, 뷰어가 해당 버퍼 참조해 렌더.
  - 뷰어: QGraphicsView 또는 OpenGL, state.rs와 동일 개념의 프레임 캐시.
- **산출물:** 공유 버퍼 연동, 캐시 기반 애니메이션 재생, 실시간 미리보기.

---

### Phase 5: [리패킹] 탭

- **기능**
  - 현재 프로젝트/데이터 셋 정보 표시.
  - `mods/` 변경 목록 및 상태 (GamePacks에 매핑 성공/실패).
  - “APK 리패킹하기” 버튼 → 진행 바 (팩 갱신 → 리스트 재생성 → 빌드 → 서명).
  - 완료 시 결과 경로(`exports/xxx-modded-YYYYMMDD.apk`), “파일 위치 열기”.
  - (옵션) ADB 연결 시 “기기에 설치/실행” 버튼.
- **구현**
  - mod_repack.py 로직을 스레드에서 실행 (또는 서브프로세스). 시그널로 진행 단계/로그 전달.
  - mods/ 디렉터리와 GamePacks 매핑 검사는 기존 `set_file`/`find_file` 활용.
- **산출물:** 리패킹 탭 UI, mod_repack 연동, 진행/결과 표시, (선택) ADB 호출.

---

### Phase 6: 지속성 및 데이터 버전 관리

- **기능**
  - 재실행 시 마지막 Workspace 로드, 최근 프로젝트 목록.
  - BCData 준비/미준비 상태 시각화 (데이터 준비 탭 또는 공통 상태바).
  - 새 게임 버전 추가: [데이터 준비]에서 버전 선택 후 동일 플로우.
  - 기존 프로젝트의 데이터 셋만 교체(업그레이드) 시 호환성 경고 메시지.
- **구현**
  - 설정 파일에 `last_workspace`, `recent_projects` 저장.
  - project.json 의 `dataset` 변경 시 “업그레이드” 플로우로 처리하고, 필요 시 백업/경고.
- **산출물:** 설정 저장/로드, BCData 상태 표시, 프로젝트 데이터 셋 업그레이드 플로우.

## 5. 비기능 요구사항

- **성능:** 애니메이션 뷰어는 프레임 캐시 사용으로 30fps 유지. 캔버스 편집 반영은 공유 버퍼로 디스크 I/O 없이 실시간.
- **호환성:** Windows 우선, 필요 시 macOS/Linux (Python + PySide6 동일 코드베이스).
- **데이터:** APK/서버 데이터는 앱에 포함하지 않고, 사용자 Workspace에만 저장.

## 8. 첫 번째 마일스톤 (M1) 권장 범위

**M1에서 “여기까지 만들기” 권장:** **Phase 0 전부 + Phase 1 전부**

- **의미:** 앱을 켜고 Workspace를 정한 뒤, **한 번에 “데이터 준비”까지 끝낼 수 있는** 최소 사용 사이클을 완성.
- **완료 시 사용자 경험:**  
  실행 → Workspace 선택(또는 자동 복원) → [데이터 준비] 탭에서 서버/버전 선택 → APK 다운로드 또는 파일 지정 → 실행 → 진행/로그 확인 → 완료 후 “이 데이터 셋으로 새 프로젝트 만들기”로 다음 단계 진입 가능.

### M1 체크리스트

| # | 항목 | 비고 |
|---|------|------|
| 1 | PySide6 앱 진입점, 단일 메인 윈도우 | 탭 4개 골격: 데이터 준비 / 프로젝트 / 스프라이트 편집 / 리패킹 (빈 위젯이라도 탭만 존재). |
| 2 | Workspace 선택 다이얼로그 | 기본값 `문서/BattleCatsSpriteLab`, 선택 시 `BCData/`, `projects/` 생성. |
| 3 | Workspace 경로 저장 및 재실행 시 자동 로드 | 설정 파일(예: `~/.config/BattleCatsSpriteLab/settings.json`) 또는 플랫폼별 적절한 위치. |
| 4 | [데이터 준비] 탭 UI | 서버(한국/일본 등), 게임 버전 선택, “자동 다운로드” / “APK 파일 지정” 선택. |

**M3에서 “여기까지 만들기” 권장:** **Phase 3 전부**

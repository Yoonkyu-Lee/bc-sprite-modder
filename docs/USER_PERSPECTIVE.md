# 유저 관점 주요 사용 플로우(요약)

## 1. 첫 실행 및 작업 환경 준비

- 프로그램 실행 시 “작업 폴더(Workspace)” 선택 다이얼로그 노출  
  - 기본값 예시: `문서/BattleCatsSpriteLab`
- Workspace 선택 후 자동 폴더 구조 생성
  - `BCData/` : APK, 서버 데이터, 캐시 저장소
  - `projects/` : 각 프로젝트 별 작업 폴더

## 2. 게임 데이터 셋업

- [데이터 준비] 탭에서 서버(한국/일본 등) 및 게임 버전 선택  
  - 최신 버전 자동 체크 및 목록 제공
- APK 자동 다운로드 또는 직접 APK/XAPK 지정 가능  
- 진행 과정:  
  1. 다운로드  
  2. APK 추출  
  3. 서버 데이터 다운로드  
  4. 팩 검사/로그 일괄 표시
- 완료 시 “이 데이터 셋으로 새 프로젝트 만들기” 버튼 제공 → 다음 단계로 이동

## 3. 프로젝트 생성 및 관리

- [프로젝트] 탭에서  
  - 프로젝트 이름/설명 입력  
  - 사용할 데이터 셋 선택 (`KR 15.1.0` 등)
- 생성 시 구조:
  - `projects/<프로젝트명>/mods/` : 수정 리소스
  - `projects/<프로젝트명>/exports/` : 결과물(APK 등)
  - `projects/<프로젝트명>/project.json` : 메타 정보
- 프로젝트 리스트에서 더블클릭하면 "현재 작업 프로젝트"로 진입

## 4. 스프라이트 탐색 & 편집

편집의 단위는 **“스프라이트 파일 하나”가 아니라 “특정 캐릭터(유닛)의 특정 폼”**입니다.  
한 폼에는 스프라이트 시트(PNG), imgcut, mamodel, maim 애니메이션 등이 묶여 있으며, 브라우저는 **캐릭터 → 폼 → 에셋** 순서로 탐색하도록 구성됩니다.

### 4.1 스프라이트 브라우저 (캐릭터 → 폼 → 에셋)

- [스프라이트 편집] 탭  
  - **1단계: 캐릭터(유닛) 선택**  
    - 유닛 ID(예: 076, 037) 또는 이름(메타 있을 경우) 목록.  
    - 광란(e)은 별도 구분 — 같은 숫자라도 e는 다른 캐릭터로 표시.
  - **2단계: 그 캐릭터의 폼 선택**  
    - 해당 유닛에 실제로 존재하는 폼만 표시: **f**(1폼), **c**(2폼), **s**(3폼), **u**(4단폼).
  - **3단계: 해당 폼의 에셋 목록**  
    - 스프라이트 시트(uni…png), 아이콘(udi…png), imgcut, mamodel, maim 애니메이션 등.  
    - 항목 클릭 시 상세 편집 화면 진입.
- 검색/필터: 유닛 ID, 폼 코드, 파일명으로 범위 좁히기.

### 4.2 편집 화면 레이아웃

- 좌상단: 애니메이션 뷰어(클립 목록, 재생/일시정지, 속도, 프레임 조절)
- 우상단: 프레임별 파트 리스트 (mamodel 파츠 ↔ imgcut cut_id 연결)
- 하단: 캔버스 에디터 (직접 그리기/지우기/레이어/외부 PNG 임포트, 투명도, 격자, 미러 등 지원)  
- **실시간 미리보기:** 저장(또는 자동저장) 시 애니메이션 뷰어가 즉시 최신 이미지로 업데이트

### 4.3 파일 관리 방식

- 모든 수정 파일은  
  - `projects/<프로젝트명>/mods/076_u.png`  
  와 같이 **폼 단위 파일명**으로 자동 정렬
- 내부데이터는  
  - 열려 있는 프로젝트/데이터셋에 따라  
  - `GamePacks.set_file(...)` 등 API로만 반영

## 5. 테스트 & APK 리패킹

- [리패킹] 탭  
  - 현재 프로젝트/데이터셋 정보, mods/ 변경내역 및 상태 표시(성공/매핑 안됨 등)
  - “APK 리패킹하기”  
  : 진행바(팩 갱신 → 리스트 재생성 → 빌드 → 서명)  
  - 완료 시:  
    - 결과 경로(`projects/<프로젝트명>/exports/xxx-modded-YYYYMMDD.apk`)
    - “파일 위치 열기”  
    - (옵션) ADB 연결 시 “기기에 설치/실행” 버튼

## 6. 장기간 사용 및 데이터 버전 관리

- 재실행 시 마지막 Workspace 자동 로드, 최근 프로젝트 목록 표시  
- BCData 준비/미준비 여부 시각화  
- 새 공식 버전 출시 시: [데이터 준비]에서 쉽게 추가 가능  
- 기존 프로젝트는 데이터셋만 교체로 ‘업그레이드’(호환성 경고 등 제공)

### 4.4 용어 (폼·파츠·imgcut)

- **폼 코드:** f(1폼), c(2폼), s(3폼), u(4단폼). **e(광란)**는 같은 숫자라도 별도 캐릭터 분류.
- **파츠(Part):** mamodel에 정의된 부위. 각 파츠는 **cut_id**로 imgcut의 한 rect와 연결됨.
- **cut_id / imgcut rect:** 스프라이트 시트 상의 사각형 영역(x, y, w, h). 파츠가 참조하는 이미지 영역.

## 전체 패턴 요약

```
1. Workspace 선택  
2. 데이터 셋 준비  
3. 프로젝트 생성  
4. 스프라이트 탐색/편집  
5. 리패킹/테스트  
```

UI 구상 예)  
좌: 애니메이션 뷰어  
우: 프레임/파트 리스트  
하단: 캔버스 에디터(실시간 연동)
이런 구조로, 자연스럽고 즉각적인 게임 애니메이션/리소스 모딩 환경을 제공합니다.
